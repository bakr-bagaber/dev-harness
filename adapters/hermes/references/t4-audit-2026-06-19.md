## Bug Found: Bare `--target` Crash in init.mjs (exit code 3)

**Root cause:** When `--target` is passed without a value, `args.flags.target` is boolean `true`. `path.resolve(true)` throws TypeError before any try/catch — propagates as exit code 3 (INTERNAL_ERROR).

**This is the second occurrence** of this bug class. First was fixed in `status.mjs` during T2 audit (bare `--target` → type guard → fallback to cwd). Same pattern was missing from `init.mjs` — the guard wasn't ported when init was built on top of status.

**Fix applied:** Add the same type guard pattern used in status.mjs:
```js
const rawTarget = args.flags?.target;
const targetDir = resolve((typeof rawTarget === 'string') ? rawTarget : process.cwd());
```

**Lesson:** Any future command that accepts `--target` must apply this guard. See `cli/commands/status.mjs` and `cli/commands/init.mjs` for the reference pattern. The `dev-harness` SKILL.md pitfalls now document this as a mandatory pattern.
