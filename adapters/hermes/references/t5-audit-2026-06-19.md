## Issues Found & Fixed

### BUG #1 (FIXED) — validateConfig reports `currentPhase: null` as missing
**File:** `cli/lib/state.mjs:387-396`
**Root cause:** `validateConfig()` treated `null` as missing for all required fields. But `currentPhase: null` is a valid value (pipeline not started) per JSON schema `type: ["string", "null"]`.
**Fix:** Added `nullable` set (`const nullable = new Set(['currentPhase'])`) that exempts `null` values from being reported as missing.

### ISSUE #2 (MINOR) — `git.hasUpstream` never updated in `transitionPhase`
`transitionPhase` updates `git.branch`, `git.clean`, and `git.lastCommitMessage` but NOT `git.hasUpstream`. Default stays `false` even when tracking upstream.
**Suggested fix:** Add `getGitUpstream()` that runs `git rev-parse --abbrev-ref --symbolic-full-name @{u}` and returns `true`/`false`.

### ISSUE #3 (MINOR) — Missing key returns `undefined`, omitted from JSON output
`resolveKey` returns `undefined` for missing keys. `JSON.stringify` omits `undefined` values, so `"value"` field disappears from `config get nonexistent.key --json`.
**Suggested fix:** Return `null` from `resolveKey()` for missing keys, or add `found` boolean.

### ISSUE #4 (DESIGN) — `init` not in default `phases.enabled`
`PHASE_ORDER` includes `init` but default config excludes it from `phases.enabled`. `harness-dev phase init` always fails. By design — `init` is CLI-only via `harness-dev init`.

### ISSUE #5 (CODE QUALITY) — Empty if block in `set()`
Lines 255-258 of state.mjs have a no-op if-block with only a comment. Logic works correctly.
