# progress.md Dual-Structure Writer (T6)

The `cli/lib/progress.mjs` module manages the dual-structure `progress.md` file: Session State (overwritten) + Lessons Learned (appended).

## File Format

```markdown
# Progress: <stackLabel>

## Session State

Current Phase: build
Current Feature: US-003
Gate Status: pending — 2 failing tests
Next Action: fix test_auth.py assertions
Retry Count: 2/3

## Lessons

2026-06-19 | Hermes | Lesson text here

## Checkpoints
```

## Key Functions

| Function | Behavior |
|----------|----------|
| `readProgress(dir)` | Parse both sections, return `{session, lessons, ok, path}` |
| `readSessionState(dir)` | Convenience — returns just `{Current Phase, Current Feature, ...}` |
| `readLessons(dir)` | Convenience — returns `[{date, author, text}]` |
| `writeSessionState(dir, fields)` | Replace `## Session State` block entirely |
| `appendLesson(dir, text, author?, date?)` | Append line to `## Lessons` section |

## Design Rules

- **Session State overwrites** — `writeSessionState` finds the `## Session State` header and replaces everything until the next `## ` header. Lessons below are preserved.
- **Lessons append** — `appendLesson` finds the last lesson line (or the `## Lessons` header) and inserts after. If no `## Lessons` section exists, it creates one.
- **Missing file handling** — All read functions return defaults with `ok: false`. `appendLesson` creates the file with minimal structure if absent.
- **Date format** — Always `YYYY-MM-DD`, generated by `new Date().toISOString().slice(0, 10)`.
- **Lesson line regex** — Parsed with `/^(\d{4}-\d{2}-\d{2})\s*\|\s*(.+?)\s*\|\s*(.+)/`.
- **Session state field regex** — Parsed with `/^(\w[\w ]+):\s*(.*)/`.

## Integration Points

- `cli/commands/learn.mjs` — Entry point for `harness-dev learn "text"`. Calls `appendLesson(targetDir, message)`.
- `cli/commands/status.mjs` — Reads lessons via `readLessons(targetDir)` for `--json` output under key `recentLessons`.

## Pitfalls

- **Learn command initially lacked `--target` support.** Same pattern as config and phase. Always add `const rawTarget = args.flags?.target; const targetDir = (typeof rawTarget === 'string') ? resolve(rawTarget) : process.cwd();` to new commands.
- **JSON output format mismatch:** The verify spec shows `lessons: ["..."]` (array of strings) but implementation returns `[{date, author, text}]` (array of objects). The richer object format is preferred for machine parsing. If agent consumers expect strings, call `.map(l => l.text)` at the consumer.
- **Lesson line insertion after lessons section is fragile:** The `appendLesson` function finds the last line matching the lesson date pattern. If the file has non-lesson lines between lessons (like comments), those break the insertion point. Keep the Lessons section clean — one lesson per line, no blank lines between entries.
