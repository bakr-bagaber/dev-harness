## Critical Bugs Fixed

### Bug #1: Rounds counter never increments (CRITICAL)

**File:** `cli/lib/contract.mjs`

**Root cause:** The regex `/rounds?:\s*\d+\/\d+/` failed to match `**Negotiation rounds:** 0/5` because `\s*` does not match the `**` (asterisk) characters after the colon. The template writes bold formatting around the round count.

**Impact:** 
- `getContractStatus()` always returned `rounds: 0`
- `reviewContract()` always computed `newRounds = 1` (0+1)
- The replacement silently failed because the regex didn't match
- Auto-escalation at 5 rounds never triggered (stuck at `newRounds=1` forever)

**Fix:**
- Parse regex: `/rounds?:\s*\*{0,2}\s*(\d+)\/(\d+)/` — added `\*{0,2}` to optionally consume `**`
- Replace regex: `/(rounds?:\s*\*{0,2}\s*)\d+\/\d+/` — capture group preserves `**` bold markers in output

### Bug #2: Auto-escalation didn't write to file (CRITICAL)

**File:** `cli/lib/contract.mjs` (reviewContract function)

**Root cause:** `reviewContract()` returned `{ escalated: true }` but only set the status to "Needs Revision" on the line that was about to be written. The `## Escalation` section was never appended and the file status remained "Needs Revision" instead of "Escalated — awaiting human adjudication".

**Fix:** When `escalated` is true, the function now:
1. Sets display status to "Escalated — awaiting human adjudication" instead of the decision value
2. Appends a `## Escalation` section with reason, rounds, and timestamp before writeFileSync

### Bug #3: Auto-escalation CLI message was misleading

**File:** `cli/commands/contract.mjs`

Before fix: `Contract needs-revision. Max rounds reached — escalated to human.`
After fix: `Max negotiation rounds reached. Contract escalated to human.` (when escalated)
           `Contract agreed.` (normal path)

### Bug #4 (cosmetic): Notes section missing trailing newline

**File:** `cli/lib/contract.mjs` (reviewContract function)

The `notesSection` replacement didn't have a trailing `\n`, causing the `## Agreement Status` header to immediately follow notes content without a blank line separator.
