## Key Discoveries

### GAP: validate --feature/--task deferred but never landed
The T7 audit (`references/t7-audit-2026-06-19.md`) flagged `--feature`/`--task` as "should be added when T8 is implemented." T8 implemented the ralph-inner engine but **did not implement the flags** in `validate.mjs`. The engine's instructions reference `harness-dev validate --feature <id> --task <id>`, but the command silently ignores those flags.

**Pattern lesson:** When you defer a feature from task A to task B, verify B actually delivers it. A "TODO for T-X" note in one audit does not guarantee T-X picks it up — cross-check the receiving task's actual file changes.

### Fix applied during audit
- `validate.mjs` now parses `--feature` and `--task` flags
- `gates.mjs` `runChecks()` accepts optional `{feature, task}` options
- JSON output includes `feature` and `task` fields when provided
- Per-feature/task filtering still unimplemented (needs task-level check functions)

### Other Spec Gaps
- Default `maxRetries: 3` in config vs spec saying "Max iterations = 10"
- Feature summary counts in config not synced with feature_list.json
- runPhase always returns `iteration: 1` (retry counting deferred to T9)
