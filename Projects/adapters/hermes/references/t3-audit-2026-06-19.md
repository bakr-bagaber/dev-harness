## Issues Found

### 1. Unguarded `main()` in templates.mjs

**File:** `cli/lib/templates.mjs`, line 221

```javascript
main();  // runs unconditionally on import
```

Importing `templates.mjs` as an ESM module (`import('./cli/lib/templates.mjs')`) triggers `main()`, which parses `process.argv` and crashes with `--stack is required` exit 2. The module's three exported functions (`substitute`, `discoverTemplates`, `generateTemplates`) are unreachable via programmatic import.

**Fix:** Guard the `main()` call with an isMain check:

```javascript
import { fileURLToPath } from 'node:url';
const __filename = fileURLToPath(import.meta.url);

// Replace: main();
// With:
const isMain = process.argv[1] && (
  basename(process.argv[1]) === 'templates.mjs' ||
  resolve(process.argv[1]) === __filename
);
if (isMain) main();
```

This matches the guard pattern used by `detect-stack.mjs` and `status.mjs`.

**Status:** ❌ UNFIXED (as of audit date)
